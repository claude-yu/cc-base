# cc-base 使用指导

本文件面向微信、飞书等聊天窗口里的实际使用。目标是少记命令、少等结果、出问题能快速恢复。

## 常用命令

| 命令 | 用途 |
|------|------|
| `/计划审查 <任务描述>` | 让 CC 写计划，并交给 Codex 审查。异步运行，返回 RunId。 |
| `/查看审查 [RunId]` | 查看计划审查进度或结果；不传 RunId 时显示最新一次。 |
| `/问codex <问题>` | 异步询问 Codex 技术建议，完成后自动回传。 |
| `/codex结果 [RunId]` | 查看 Codex ask 结果；不传 RunId 时显示最新一次。 |
| `/修复 <问题描述>` | CC 直接修复 controller/cc-connect 报错。等价于 `/修复controller`。 |
| `/md状态检查 [MD工作目录]` | 只读检查任意 MD/GROMACS 目录，列文件并读取 log tail。 |
| `/质询计划` | 启动 Grill-Me，对最近计划逐条质询。 |
| `/学习状态` | 查看 chat-instinct 观察记录和习惯统计。 |
| `/进化习惯` | 分析观察记录，生成习惯进化候选。 |
| `/自动回传 开` | 开启计划审查完成后自动推回聊天窗口。 |
| `/自动回传 关` | 关闭计划审查自动回传，改为手动查看。 |

## 推荐交互方式

### 计划类任务

先用：

```text
/计划审查 帮我设计 xxx 的实现方案
```

如果返回 REVISE，继续用：

```text
/质询计划
```

当计划通过后，再按提示执行 `/批准执行 <RunId>` 或 `/人工批准执行 <RunId>`。

### Controller 或聊天链路报错

用：

```text
/修复 报错内容
```

适合处理：
- PowerShell parser error
- 中文乱码
- cc-connect config 同步问题
- 回调没有发送
- command 参数传递错误
- controller 脚本路径或权限问题

`/修复` 只应该处理 controller/cc-connect 基础设施，不用于修改科研项目数据。

### MD/GROMACS 状态检查

检查指定目录：

```text
/md状态检查 E:\path\to\gromacs_run
```

不传路径时检查 `CC_WORK_DIR`：

```text
/md状态检查
```

该命令只读扫描 `.tpr/.cpt/.log/.xtc/.trr/.edr/.gro/.pdb/.top/.itp/.mdp`，不会运行 GROMACS。

### Codex 建议

用：

```text
/问codex 你觉得这个方案有什么风险？
```

它会立即返回 RunId，后台完成后自动回传。若没看到回传，用：

```text
/codex结果
```

或：

```text
/codex结果 <RunId>
```

### 自动回传

默认建议开启：

```text
/自动回传 开
```

如果不希望计划审查完成后主动推送，关闭：

```text
/自动回传 关
```

查看状态：

```text
/自动回传
```

## 学习与进化

查看当前观察记录：

```text
/学习状态
```

分析重复习惯并生成候选：

```text
/进化习惯
```

原则：
- chat-instinct 记录命令使用模式和重复修复模式。
- 进化候选需要用户确认后才应该写成 instinct/skill。
- 不把一次性偏好直接固化，避免噪音污染长期记忆。

## 什么时候说“go on”

优先使用明确命令查看状态：

```text
/查看审查
/codex结果
/自动回传
```

只有在普通 CC 对话中已经明确有一个等待点时，再说“继续”或“go on”。对于计划审查和 Codex ask，优先用 RunId 查询，结果更稳定。
